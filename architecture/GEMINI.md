# GEMINI.md: The Sovereign Data Schema

## 🏛️ Purpose
This file acts as the technical source of truth for the Skills Nexus. It defines the structure of our library telemetry and the "Technical Sovereignty" of the Antigravity Skills Master.

## 📊 Data Mapping (Handshake.json)
The `Handshake.json` file generated by `metadata_scraper.py` must follow this schema:

### 1. Domain Object
```json
{
  "name": "string",
  "path": "string",
  "health_score": "number",
  "skills_count": "number",
  "description": "string"
}
```

### 2. Skill Object
```json
{
  "name": "string",
  "domain": "string",
  "path": "string",
  "is_artifact": "boolean",
  "last_audit": "ISO8601"
}
```

## 🛠️ Operational Rules
1. **No Ad-hoc Data**: All UI data must originate from the filesystem or NotebookLM.
2. **A.N.T. Strictness**: Components must read SOPs from `architecture/` before implementing features.
3. **Self-Healing**: If `Handshake.json` is missing or malformed, the application must attempt to re-run the scraper tool.
